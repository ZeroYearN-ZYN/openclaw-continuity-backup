## ⚙️ 配置说明

### 修改搜索关键词
编辑 `scripts/pubmed_fetch.py`：
```python
SEARCH_TERMS = [
    "CHO cells protein expression",
    "Chinese hamster ovary recombinant",
    "CHO cell culture production",
    # 添加新关键词...
]
```

### 修改抓取时间范围
```python
DAYS_BACK = 30  # 抓取最近30天的文献
```

### 修改并发下载数
编辑 `scripts/download_pdfs_enhanced.py`：
```python
MAX_WORKERS = 3  # 同时下载3个文件
TIMEOUT = 30     # 超时时间（秒）
```

## 💡 使用建议

### 首次运行建议
1. **先测试**: 手动运行一次 `pubmed_fetch.py` 确认配置正确
2. **检查结果**: 查看JSON文件是否包含你需要的文献
3. **下载PDF**: 运行下载器，看看开放获取的比例
4. **调整关键词**: 根据结果优化搜索词

### 日常使用
- **每天8:00** 自动运行
- **查看桌面** `CHO-Literature/日期/` 文件夹
- **文献数量** 因搜索词和发表周期波动，这是正常的
- **PDF下载** 只有开放获取的文献能下载，其他需手动

### 优化建议
1. **关键词精炼**: 太宽泛→太多无关文献；太窄→漏掉相关文献
2. **定期清理**: 删除旧的JSON和PDF，节省空间
3. **API限制**: 如遇频繁错误，可能触发了API限制，等待几小时
4. **网络问题**: PDF下载失败可能是网络波动，重新运行即可

## ❓ 常见问题

### Q: 为什么很多PDF下载失败？
A: 大多数学术文献不是开放获取。Unpaywall能下载约20-30%的免费PDF，其他需要：
- 机构订阅
- 通过DOI链接手动下载
- 联系作者获取

### Q: Semantic Scholar为什么不用了？
A: Semantic Scholar有更严格的API限制（429错误）。PubMed足够且更稳定。

### Q: 可以下载其他主题的文献吗？
A: 当然！修改 `SEARCH_TERMS` 即可，比如：
- `"CRISPR gene editing"`
- `"machine learning drug discovery"`
- 任何PubMed支持的关键词

### Q: 如何获取更多PDF？
A: 
1. 使用机构VPN（如果有的话）
2. 通过图书馆访问
3. 使用Sci-Hub（注意版权）
4. 直接邮件联系作者
